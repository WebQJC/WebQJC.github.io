前端引入icon的方法

本地已经有仓库使用git提交项目到GitHub

  • 1、$ git init
  • 2、$ vim .gitignore（作用是过滤不需要上传的文件）
  • 3、$ git status（可以看到已经被修改的文件或上传文件）
  • 4、$ git add . （后面有一个点，意思是将你本地所有修改了的文件添加到暂存区）
  • 5、$ git commit -m ""（引号里面是你的介绍）
  • 6、$ git remote add origin （后面加远程链接，这个origin可以自己取名）
  • 7、$ git push -u origin master（将代码推至远程就可以了）

使用clone拷贝代码到本地,修改代码后推送

  • 1、$ git init
  • 2、$ git clone（后面加远程链接；或者 命令：git clone -b 分支名 git地址）
  • 3、cd 文件名
  • 4、$ git add . （后面有一个点，意思是将你本地所有修改了的文件添加到暂存区）
  • 5、$ git commit -m ""（引号里面是你的介绍）
  • 6、$ git push（将代码推至远程就可以了）

分支

  • 1、$ git init
  • 2、$ git clone（后面加远程链接；或者 命令：git clone -b 分支名 git地址）
  • 3、cd 文件名
  • 4、$ git checkout -b dome（创建分支并切换到分支）
  • 5、$ git add . （后面有一个点，意思是将你本地所有修改了的文件添加到暂存区）
  • 6、$ git commit -m ""（引号里面是你的介绍）
  • 7、$ git checkout master （切换到master分支）
  • 8、$ git merge dome（合并分支）
  • 9、$ git log（查看提交的commit）
  • 10、$ git branch -d dome （删除分支）
  • 11、$ git branch（查看分支）

下拉远程代码和你本地的代码合并

  • 1、$ git init
  • 2、$ git stash（这是将本地代码回滚值至上一次提交的时候，就是没有你新改的代码）
  • 3、$ git pull origin master（将远程的拉下来）
  • 4、$ git stash pop （将第一步回滚的代码释放出来，相等于将你修改的代码与下拉的代码合并）
  • 5、$ git add . （后面有一个点，意思是将你本地所有修改了的文件添加到暂存区）
  • 6、$ git commit -m ""（引号里面是你的介绍）
  • 7、$ git push（将代码推至远程就可以了）
  • 8、建议最后再git pull origin master 一下，确保远程的全部拉下来，有的你刚提交完有人又提交了，你再拉一下会避免比的不是最新的问题
